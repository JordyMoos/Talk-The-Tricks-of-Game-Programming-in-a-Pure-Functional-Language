Elm is a bit different then most of the languages out there.
Which sometimes scare people off.
But I actually find Elm one of the easiest languages to learn.
  • Pure Functional language
  • Statically Typed
  • Strongly Typed
Some of the characteristics of Elm is that it is a Pure Functional language, Its statically and stronly typed.
Which allows the compiler to complain when I'm doing things wrong again
But its complains are often so clear that you can learn a lot from the compiler
A simple function in elm looks like this
We have the name of the function `add`
The arguments `x` and `y`
And then on the right side of the equal sign is the body of the function
Now everything in Elm is typed and you can annotate the function with its types
Here we say the type of `add` is. We take a number, and a number.
And then we will return a number.
Applying functions is done by spaces.
And function arguments are also separated by spaces
Therefor we call `add` like this
Everything in Elm is curried. Therefor you can call a function with less then all arguments
And you will get back a function that requires the remaining arguments.
Calling `add` with just `5` will give us a function that requires another argument before computing a result
So far Elm is not that different. At least to languages like Haskell.
It is compared to javascript hehe
What Elm also has is some kind of architecture
In Elm you always have a Model, which is the state of your application
update which is a function that creates new states
And a view. Which is a function to visualize your state.
In this case we define to be just an integer
To update your state, or to create a new model, you need to have some intend
These you define as messages in Elm.
This example is a little counter which you can increment and decrement
A produced message together with your model will be given to the defined `update` function.
Which in return will return a new model
As we can see in the type of the update function as well.
Given a msg and a model will return a model
To visualize your state. You have the view function.
Which received the model and returns some sort of html.
To visualize the architecture in one picture
The puppets and their orientation is your Model
The theater is the result of the view
The kids are producing messages
Which Mister `Update` received and it adjust the model as requested
